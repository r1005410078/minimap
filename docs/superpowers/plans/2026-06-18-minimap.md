# Minimap 功能任务拆分

## 说明

这份文档先按功能拆任务，不进入实现代码。每个任务都应该可以独立验收，并且尽量先完成可测试的纯逻辑，再接 Canvas 和 Vue 交互。

## 任务 1：图数据模型和示例数据

**目标：** 建立 minimap 的基础 graph 数据结构，支撑普通节点、父子关系、分组、排序和大数据生成。

**包含功能：**

- 定义节点字段：`id`、`label`、`parentId`、`children`、`kind`。
- 定义边字段：`id`、`source`、`target`、`label`、`kind`、`data`。
- 定义 graph 版本字段：`version`。
- 支持节点尺寸字段：`width`、`height`，并为 `measureNode` 预留输入。
- 创建能源系统示例数据。
- 创建 10000 节点压力测试数据。
- 支持从资源树新增节点。
- 支持分组框内部子节点顺序调整。
- 支持删除、复制、导入、导出所需的数据操作接口预留。

**验收点：**

- 能生成示例 graph。
- 能生成 10000 节点 graph。
- 能表达父子关系之外的业务连接线。
- 调整节点顺序后，真实 graph 数据顺序发生变化。

## 任务 2：布局引擎

**目标：** 用纯 JavaScript 实现稳定分层树布局，支持左右布局和上下布局，不依赖 Vue 或 DOM。

**包含功能：**

- 从 `rootIds` 开始遍历父子树。
- 在布局前把超过阈值的连续兄弟节点折叠成分组项。
- 自底向上计算节点、分组框和子树占用尺寸。
- 自顶向下分配节点和分组框的世界坐标。
- 左右布局坐标计算。
- 上下布局坐标计算。
- 节点、连线、分组框的包围盒计算。
- 根据 `node.width/height`、`theme.nodeSize` 或 `measureNode(node)` 计算节点尺寸。
- 内容整体边界计算。
- 布局切换前后的目标坐标计算。
- 视口锚点补偿计算，避免重新布局后用户丢失当前操作位置。
- 世界坐标、屏幕坐标和视口坐标的转换函数。
- `edges` 默认不参与主布局，只参与绘制、关系高亮和搜索定位。

**验收点：**

- 同一份 graph 可以生成左右和上下两种布局。
- 父子节点方向符合布局模式。
- 父节点位于子树或子分组的中线位置。
- 兄弟节点顺序在布局结果中保持稳定。
- 额外业务关系线不会改变父子树主布局。
- 布局切换后可以根据选中节点计算视口补偿。
- 坐标转换可以被拖入、缩放、框选和 overview 复用。

## 任务 3：自动分组和分组框布局

**目标：** 实现相邻节点超过 5 个后自动合并到分组框，并支持分组框内部滚动和虚拟绘制。

**包含功能：**

- 相邻兄弟节点检测。
- 超过 5 个连续兄弟节点时生成分组框。
- 支持通过 `options.groupThreshold` 配置分组阈值。
- 为后续 `shouldGroup` 或 `groupBy` 自定义分组策略预留接口。
- 分组框最大宽高按视口比例计算。
- 分组框内部网格布局。
- 分组框内部滚动区域计算。
- 只返回当前可见的分组子节点。
- 支持分组展开和折叠状态。

**验收点：**

- 5 个及以下节点不自动合并。
- 6 个及以上相邻节点自动合并为分组框。
- 调整分组阈值后，自动分组结果同步变化。
- 大分组不会一次性绘制所有子节点。
- 滚动后可见子节点集合变化。

## 任务 4：Canvas 渲染器

**目标：** 用 Canvas 绘制大图主体，并保持 10000 节点场景下的可用性能。

**包含功能：**

- 绘制网格背景。
- 绘制普通节点。
- 绘制连线。
- 绘制分组框。
- 绘制分组框内部可见子节点。
- 支持普通节点、分组框、连线的自定义绘制函数。
- 为自定义绘制函数提供统一参数：业务数据、包围盒、交互状态、主题、视口。
- 绘制选中态、多选态、拖拽态。
- 绘制自定义滚动条。
- 根据视口裁剪，只绘制可见元素。
- 收集渲染耗时和可见元素数量。
- 处理 DPR，保证高清屏下 Canvas 不模糊。
- 响应容器 resize。

**验收点：**

- 示例图能正确显示。
- 外部传入自定义节点绘制函数后，节点视觉能被替换。
- 10000 节点压力图不会创建 10000 个 DOM 节点。
- 缩放和平移时画面连续，没有明显跳动。
- 容器尺寸变化后，Canvas 尺寸和内容绘制同步更新。

## 任务 4.5：组件契约接口

**目标：** 明确 minimap 作为可复用组件时对外暴露的 props、events 和 methods。

**包含功能：**

- `graph`、`resources`、`layoutDirection`、`readonly` 等基础 props。
- `loading`、`error` 等状态 props。
- `selectedIds`、`groupStates`、`viewport` 等受控状态 props。
- 受控/非受控状态规则。
- `nodeRenderer`、`groupRenderer`、`edgeRenderer` 自定义绘制 props。
- `measureNode` 节点尺寸测量 prop。
- `theme` 和 `options` 配置 props。
- `change`、`select`、`viewport-change`、`layout-change` 等状态事件。
- `node-drop`、`node-move`、`group-reorder`、`group-state-change` 等编辑事件。
- `beforeNodeDrop`、`beforeNodeMove`、`beforeGroupReorder`、`beforeDelete`、`beforeCopy` 等拦截钩子。
- `fitToScreen()`、`centerOnNode()`、`zoomTo()`、`select()`、`undo()`、`redo()` 等命令式方法。
- `resize()` 和 `destroy()` 生命周期方法。

**验收点：**

- 外部可以用 props 控制布局方向、选中状态、视口和只读模式。
- 外部可以监听图数据变化、选择变化、视口变化和分组换位。
- 外部可以在拖入、移动、换位、删除、复制前阻止默认行为。
- 外部可以通过组件实例调用 fit、center、zoom、select、undo、redo 等方法。
- 自定义绘制函数不需要访问组件内部私有状态。

## 任务 5：视口控制

**目标：** 实现平移、缩放、适配视图、定位选中和 overview 小地图。

**包含功能：**

- 空白区域拖拽平移。
- 鼠标滚轮或触控板缩放。
- 以鼠标位置为中心缩放。
- 最小/最大缩放比例限制。
- fit to screen。
- center selected。
- overview 小地图绘制。
- overview 视口框拖拽导航。
- DPR 和容器 resize 后的视口同步。

**验收点：**

- 缩放时鼠标下方的世界坐标保持稳定。
- fit to screen 后全图进入视口。
- 点击或拖动 overview 后，主画布视口同步变化。
- resize 后当前视口不丢失。

## 任务 6：选择系统

**目标：** 支持单选、多选、框选和选择关系高亮。

**包含功能：**

- Canvas 命中检测。
- 单选节点或分组。
- `Shift`、`Cmd`、`Ctrl` 多选。
- 框选可见节点。
- 点击空白区域取消选择。
- `Esc` 取消选择。
- `Delete` 删除选中。
- 方向键微调选中节点。
- `Cmd/Ctrl+A` 选择可见节点。
- `Cmd/Ctrl+Z` 撤销，`Cmd/Ctrl+Shift+Z` 或 `Cmd/Ctrl+Y` 重做。
- `+/-` 缩放。
- 选中节点后高亮父级、子级和相关连线。

**验收点：**

- 单击节点只保留一个选中项。
- 多选快捷键可以追加或取消选择。
- 框选矩形内的可见节点被选中。
- 选中节点的关系路径可被识别并高亮。
- 基础键盘快捷键可以通过配置开启或关闭。

## 任务 7：拖拽和编辑

**目标：** 支持资源树拖入、普通节点拖动、分组框内部换位，以及基础编辑操作。

**包含功能：**

- 左侧资源树原生拖拽。
- 资源拖入 Canvas 后新增节点。
- 普通节点拖动。
- 分组框内部节点拖拽换位。
- 拖拽边界约束。
- 是否允许跨父级移动。
- 是否吸附网格。
- 是否允许拖进分组框。
- 删除选中节点。
- 复制选中节点。
- 只读模式下禁用编辑操作。

**验收点：**

- 从资源树拖入后 graph 增加节点。
- 分组框内部换位后 graph 顺序变化。
- 只读模式下拖入、删除、换位都不会生效。
- 拖拽约束配置会影响移动结果。

## 任务 8：历史记录

**目标：** 为关键编辑操作提供撤销和重做能力。

**包含功能：**

- 记录节点拖入。
- 记录分组框内部换位。
- 记录节点删除。
- 记录节点复制。
- 记录布局方向切换。
- 支持撤销。
- 支持重做。

**验收点：**

- 执行编辑后可以撤销回上一步。
- 撤销后可以重做。
- 新操作发生后，重做栈被清空。

## 任务 9：搜索和定位

**目标：** 让用户在大图中快速找到节点。

**包含功能：**

- 按 `label` 搜索。
- 按 `id` 搜索。
- 搜索结果列表。
- 点击搜索结果后选中并居中节点。
- 多个结果时支持上一个/下一个。
- 搜索结果和当前状态通过 aria 状态区域可读。

**验收点：**

- 可以搜索到普通节点和分组框内部节点。
- 跳转后目标节点进入视口中心附近。
- 搜索结果和当前选中状态同步。

## 任务 10：应用壳和工具栏

**目标：** 把 minimap 能力组织成一个可试用的页面。

**包含功能：**

- 左侧资源树。
- 顶部或画布内工具栏。
- 布局切换按钮。
- 缩放按钮。
- fit to screen 按钮。
- center selected 按钮。
- 搜索入口。
- 撤销/重做按钮。
- 只读/编辑模式切换。
- 压力测试开关。
- 性能状态展示。

**验收点：**

- 页面首屏就是可操作的 minimap，不做营销式落地页。
- 常用操作可直接触达。
- 10000 节点模式可以一键切换。

## 任务 10.5：状态、错误和可访问性

**目标：** 补齐组件在真实业务场景中的基础状态和最低可访问性。

**包含功能：**

- loading 状态。
- 空 graph 状态。
- 导入失败、数据非法和渲染异常错误状态。
- `error` 事件。
- 画布容器可聚焦。
- 当前选中数量、选中节点 label、搜索结果和错误信息通过 aria 状态区域可读。
- 组件销毁时取消动画帧、移除事件监听、断开 ResizeObserver。

**验收点：**

- loading、empty、error 三类状态都能展示。
- 键盘可以聚焦画布并触发基础快捷键。
- 销毁组件后不再保留全局事件监听和动画循环。

## 任务 11：测试和验收

**目标：** 用单元测试覆盖核心纯逻辑，用轻量 E2E 或浏览器手动验收覆盖真实交互和性能。

**包含功能：**

- 图数据生成测试。
- 分组阈值测试。
- 分组框尺寸和滚动测试。
- 左右/上下布局测试。
- 视口锚点补偿测试。
- 选择逻辑测试。
- 撤销/重做测试。
- 坐标转换测试。
- 边数据和关系高亮测试。
- before hooks 拦截测试。
- resize 和 DPR 手动验证。
- loading、empty、error 状态验证。
- 生产构建验证。
- 浏览器手动验证缩放、框选、多选、拖入、换位和 overview。
- E2E 冒烟测试：页面打开后 Canvas 非空。
- E2E 冒烟测试：切换左右/上下布局后 Canvas 仍有内容。
- E2E 冒烟测试：滚轮缩放后 viewport scale 变化。
- E2E 冒烟测试：拖动画布后 viewport x/y 变化。
- E2E 冒烟测试：框选后选中数量变化。
- E2E 冒烟测试：资源树拖入后节点数量变化。
- E2E 冒烟测试：开启 10000 节点压力模式后页面仍然响应。

**验收点：**

- 核心逻辑测试通过。
- 如果允许新增开发依赖，则使用 Playwright 自动执行轻量 E2E。
- 如果不允许新增任何测试依赖，则保留同等范围的手动浏览器验收清单。
- 生产构建通过。
- 示例数据和 10000 节点数据都能在浏览器中打开并操作。
